# PROFILE_SETUP

## Overview

This repository is the special GitHub profile repository for `ghostlucius/ghostlucius`.

The profile is built from:

- A hand-written `README.md`
- A generated local SVG activity graph in `assets/activity-graph.svg`
- One GitHub Actions workflow that refreshes dynamic sections
- One local Node.js script: `scripts/update-profile.mjs`

The dynamic sections are text-based and repository-owned. They do not depend on third-party README card images for statistics or featured repositories.

### Statistics

Generated by `scripts/update-profile.mjs`.

This section updates:

- public repository count
- follower count
- following count
- account creation date
- top public repository languages by repository count
- local activity graph image

The activity graph is generated into:

- `assets/activity-graph.svg`

This avoids external stats cards and keeps the graph inside the repository.

### Recent Activity

Generated by `scripts/update-profile.mjs` from the GitHub public events API.

It shows recent public actions such as:

- pushes
- pull requests
- issues
- releases
- repository or branch creation events

### Latest Repositories

Generated by `scripts/update-profile.mjs` from the GitHub API.

It shows the latest public repositories by recent push activity, excluding:

- the profile repository itself
- repositories already listed in Featured Projects

## Troubleshooting

### Statistics are not updating

Check:

- the `profile-refresh.yml` workflow ran successfully
- `GITHUB_TOKEN` permissions include `contents: write`
- the workflow file is on the default branch

### Recent Activity is empty

Possible causes:

- there is no recent public activity
- the GitHub public events feed returned no supported events
- the workflow failed

### Latest Repositories is empty

Possible causes:

- there are no public repositories outside the featured set
- the API call failed

### Activity graph image is missing

Check:

- `assets/activity-graph.svg` exists in the repository
- `profile-refresh.yml` completed successfully
- the README references `./assets/activity-graph.svg`

### Workflow committed nothing

That usually means:

- the generated content did not change
- the section output matched the current README

## Audit Notes

### Permissions

Each workflow is limited to:

- `contents: write`

No broader permissions were added.

### Secrets handling

- No secrets are committed to the repository
- Secrets are only read from GitHub Actions

### Broken-link strategy

The profile avoids external repository/stat cards for core statistics and featured projects.

Remaining external dependencies are limited to:

- GitHub-hosted badge images

### Accessibility

- Every image in `README.md` has alt text
- Dynamic content is primarily text-based
- The activity graph SVG includes a `<title>` and `<desc>`

### Dark/light compatibility

- The local activity graph uses a transparent background
- Colors are chosen to remain readable on both dark and light GitHub themes
- The README layout does not depend on dark-mode-only screenshots or banner graphics
